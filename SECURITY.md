# Security policy

Deep Work has not released a supported runtime version. Please do not place
vulnerability details, credentials, tenant data, provider transcripts, or private
repository content in a public issue.

Report suspected issues with
[GitHub private vulnerability reporting](https://github.com/spencerthomas/deepwork/security/advisories/new).
If that mechanism is unavailable, do not publish exploit details; contact the
repository owner privately to establish a safe channel.

There are currently no released, supported runtime versions. Security fixes are
made on the default branch until a versioned support policy is published.

The product security architecture and current boundaries are documented in
[docs/SECURITY.md](docs/SECURITY.md).
