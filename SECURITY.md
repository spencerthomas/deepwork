# Security policy

Deep Work has not released a supported runtime version. Please do not place
vulnerability details, credentials, tenant data, provider transcripts, or private
repository content in a public issue.

A private reporting address and supported-version matrix must be established before
public launch. Until then, report suspected issues privately to the repository
owner through the hosting platform's private security-reporting mechanism when it
is enabled. If no private mechanism is available, do not publish exploit details;
contact the maintainer privately to establish a safe channel.

The product security architecture and current boundaries are documented in
[docs/SECURITY.md](docs/SECURITY.md).
