create_graph requires an injected BaseChatModel and accepts optional LangChain
tools plus an optional caller-owned checkpointer. Its public flow is plan,
LangGraph approval interrupt, Deep Agents execution, rejection, or bounded
respond/replan, and terminal output. validate_approval_response rejects malformed
resume values before Command construction. validate_plan_edit bounds plan state
before the public update_state(..., as_node="plan") plus re-invoke sequence. The
default checkpointer is in-memory only.

create_journey_graph runs the same plan-first flow around a research_profile or
writing_profile bundle: an executor prompt, an immutable RubricSpec version with
identified criteria and a max_iterations bounded by
HARD_VERIFICATION_ITERATION_CAP, and declarative synchronous subagents that
carry only name, description, and instructions. The executor is the pinned
public create_deep_agent with response_format=JourneyArtifactDraft; the beta
RubricMiddleware (deepagents 0.6.12, VERIFIER_REF) grades, repairs, and stops at
the cap; profile subagents run through the task tool of the synchronous
SubAgentMiddleware stack that create_deep_agent assembles. validate_artifact
bounds the declared artifact and keeps the evidence/inference/unverified claim
basis distinction; unsupported citations are downgraded and recorded in
citation_flags. The verification record preserves the full ordered verdict
history including failed iterations, and journey_capabilities reports hosted
artifact storage/transfer, async subagents, and the coding journey as
unavailable.

Importing the package performs no network call, reads no environment variable,
and does not create a model, graph, provider client, deployment, application
session, or durable persistence. runtime_capabilities reports only
local-runtime availability, required model injection, and unavailable managed
external providers.
